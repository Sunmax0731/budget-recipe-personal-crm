# 競合ベンチマーク

        ## 比較対象

        - Notion: 採用基準は入力負荷、記録粒度、証跡性、ローカル完結性。
- Obsidian: 採用基準は入力負荷、記録粒度、証跡性、ローカル完結性。
- Microsoft To Do: 採用基準は入力負荷、記録粒度、証跡性、ローカル完結性。
- レシピ管理アプリ: 採用基準は入力負荷、記録粒度、証跡性、ローカル完結性。

        ## 採用する基準

        - 入力負荷が低い。
        - 次アクションが明確。
        - ローカルまたは閉域で検証できる。
        - docs と release evidence から状態を追える。

        ## 差別化

        固定費、レシピ在庫、連絡先、次アクションをローカルで束ねる個人CRMに絞り、汎用ノートや大型サービスよりも閉域アルファの検証可能性を優先します。
